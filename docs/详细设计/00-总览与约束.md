## 2. 目标与非目标
### 2.1 目标
- 形成可直接指导实现的详细设计文档集，避免实现阶段再做关键决策。
- 明确 V1 产品边界、运行约束、默认值、接口契约和数据模型。
- 固化后续变更流程：任何实现改动必须先更新对应设计文档。

### 2.2 非目标
- 当前阶段不输出可运行代码。
- 当前阶段不执行数据库迁移、不生成 SQL 脚本文件。
- 当前阶段不引入 V1 范围外供应商（腾讯云、华为云、AWS）。

## 3. V1 产品边界
### 3.1 功能范围
- 供应商账号管理：阿里云、Cloudflare。
- 域名与解析记录管理：查询、同步、增删改查。
- DDNS 任务管理：创建、暂停、恢复、单次执行、周期执行。
- 系统日志（按类型区分）：查询、按保留策略自动清理。
- 管理后台鉴权：内置单管理员账号与会话管理。

### 3.2 明确边界
- 部署拓扑：单实例，不支持多实例并发调度。
- 鉴权模型：单管理员账号（非多租户、非 RBAC）。
- Cloudflare 凭证：仅支持 API Token，不支持 Global API Key。
- 公网 IP 获取：仅外部 HTTP 源，不使用网卡探测。

## 4. 全局约束与默认值
| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| `sync_ttl_sec` | `600` | 查询域名/记录时，缓存超过 10 分钟触发阻塞同步。 |
| `ddns.default_interval_sec` | `300` | DDNS 任务默认轮询周期。任务可覆盖。 |
| `ddns.retry_delays_sec` | `[2,4,8]` | 单次执行内部重试间隔。 |
| `logs.retention_days` | `90` | 系统日志保留 90 天（`ddns_task` / `public_ip_check` / `operation`）。 |
| `ip_sources` | `api.ipify.org`, `checkip.amazonaws.com`, `ifconfig.co/ip` | 仅外部 HTTP 获取公网 IP。 |
| `auth.session_ttl_sec` | `43200` | 管理员会话有效期，默认 12 小时。 |
| `auth.force_change_password` | `true` | 首次登录后必须修改密码。 |

## 5. 非功能设计
### 5.1 安全性
- 供应商凭证采用 `AES-256-GCM` 加密后存储于数据库。
- 主密钥来源：环境变量 `LITEDNS_MASTER_KEY`（Base64 编码，解码后 32 字节）。
- 若主密钥缺失或格式错误，服务启动失败并输出明确错误。
- 会话令牌仅存储哈希值，不落库存明文 token。
- 首次启动自动生成随机管理员密码，仅启动日志输出一次。

### 5.2 可用性
- Provider API 调用失败时采用有限重试并记录失败原因。
- DDNS 单任务失败不影响其他任务执行。
- 查询同步失败时返回错误并保留旧缓存，不写入半状态数据。

### 5.3 性能
- 查询优先读取本地缓存；仅在过期时触发阻塞同步。
- 同步与 DDNS 请求设置超时，防止长时间阻塞 goroutine。
- SQLite 索引覆盖常用查询路径（按供应商、域名、任务状态、时间范围）。

### 5.4 可观测性
- 系统日志以 `type` 区分事件来源：`ddns_task`、`public_ip_check`、`operation`。
- `operation` 类型记录管理操作（谁、何时、对何对象、结果如何）。
- `ddns_task` / `public_ip_check` 类型记录执行结果、IP 变化、错误信息、耗时。
- 健康检查接口用于快速确认服务状态与依赖状态。

### 5.5 备份与恢复
- SQLite 文件级备份策略：建议每日备份 `data.db`，保留最近 7 至 30 个版本。
- 恢复步骤：停服务 -> 替换 `data.db` -> 启动并执行一次全量同步校验。
- 主密钥恢复要求：备份恢复必须使用同一主密钥或密钥轮换后的可解密链路。

## 6. 运行配置模型
### 6.1 配置来源优先级
1. 环境变量
2. `config.yaml`
3. 内置默认值

### 6.2 启动期必须校验项
- `LITEDNS_MASTER_KEY` 可用且长度合法。
- SQLite 文件可读写，目录权限正确。
- 默认管理员是否已初始化（若无则执行初始化流程）。
- 基础配置合法（端口、同步阈值、重试参数、日志保留天数）。

### 6.3 热更新边界
- 可热更新：`sync_ttl_sec`、日志保留天数、任务默认周期、公网 IP 检查开关、公网 IP 检查间隔、IP 源列表。
- 不可热更新：主密钥、监听端口、数据库路径、加密算法。

## 7. 变更管理机制（强制）
### 7.1 原则
- 任何需求、接口、数据结构、默认值变化，必须与设计文档同提交更新。
- 审核顺序：先审文档，再审实现。

### 7.2 最小更新集
- 全局边界/默认值/非功能变化：更新本文档。
- 流程/调度/降级变化：更新 `10-系统架构与流程.md`。
- API/字段/表结构变化：更新 `20-API与数据模型.md`。

### 7.3 PR 准入规则
- 若代码涉及接口/模型/流程而文档未更新，PR 不允许合并。
- 每次文档更新必须包含：`变更摘要`、`影响范围`、`兼容性结论`。
